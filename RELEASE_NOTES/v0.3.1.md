# MCPAP Vendor Map — Patch v0.3.1

Release date: 2025-10-26
QA: Manual QA completed by user

## Highlights

- **Four-Tier Badge System**: Expanded from 2 to 4 vendor badge types with hierarchy (Event Staff > Partner > Featured > Returning)
- **Badge Hierarchy**: Only highest priority badge shown on map per booth, all earned badges displayed in panel
- **Enhanced Badge UI**: 2x2 grid panel layout with hover tooltips, collapsible legend, improved zoom-based visibility
- **Search Enhancement**: Booth IDs now searchable in vendor list (B01, B02, etc.)
- **Scheduled Days Improvements**: Viewer-friendly display with date formatting, hidden checkboxes for viewers
- **Google Places Autocomplete Integration**: Smart address validation and prediction added to search box and business address fields
- **Location Markers**: Interactive pin markers for searched addresses with click-to-remove functionality
- **Business Address Management**: New business address field with admin editing and viewer read-only access
- **Network Accessibility**: Application now accessible from any network interface (0.0.0.0:5000)
- **Enhanced Address UX**: Two-line mailing format display with map navigation and copy-to-clipboard features

## Badge System Enhancements

### Four Badge Types with Hierarchy
- **Event Staff** (Highest Priority): Red ribbon with person icon
- **Partner Vendor**: Blue ribbon with faceted diamond icon
- **Featured Vendor**: Gold ribbon with star icon
- **Returning Vendor** (Lowest Priority): Teal ribbon with circular arrow icon

### Badge Display Rules
- **Map Display**: Only highest priority badge shown per booth polygon
  - Badge hierarchy: Event Staff > Partner > Featured > Returning
  - Appears at top-right corner of booth polygon at zoom ≥20.75
  - Ribbons use semi-transparent backgrounds (28% opacity) with icon overlay
- **Panel Display**: All earned badges shown in 2x2 grid layout
  - 40px × 40px badges with hover effects (scale 1.08, brightness 1.15)
  - Tooltips on hover showing badge type
  - Side-by-side layout with logo preview (170×170px logo, 110px badge container)
  - Hidden state for badges not earned by vendor

### Badge Management
- **Admin Controls**: Visual checkbox selectors with badge icons in 4-column grid
  - Checkboxes with icon previews matching map appearance
  - Hover states and checked indicators
  - Instant save on toggle with automatic redraw
- **Shared SVG Definitions**: Single source of truth for badge artwork using `<use>` elements
  - Reduces code duplication across map badges, panel badges, and legend
  - Consistent rendering and easy theme updates

### Badge Visibility Fixes
- **Zoom-Based Rendering**: Fixed three separate issues causing badges to appear at wrong zoom levels
  - Badge creation now defaults to hidden, checks zoom before showing
  - `bringShapeToFront()` respects zoom level when setting visibility
  - `updateOverlapVisibility()` checks zoom threshold before displaying badges
- **Z-Index Hierarchy**: Labels (100) < Logo badges (200) < Status badges (300)
- **DomOverlay Integration**: All badges use `setVisible()` API (no `setMap()` fallback)

### Collapsible Badge Legend
- **Legend Location**: Bottom-right of map, visible in both admin and viewer modes
- **Collapse Feature**: Toggle button with smooth animation (0.25s transition)
  - Saves state to localStorage
  - Chevron rotates -90° when collapsed
  - Compact 12px padding when collapsed
- **Visual Design**: Semi-transparent dark background with backdrop blur
  - All four badge types listed with icons and labels
  - 180px width with consistent gap spacing

## Address Search & Location Features

- **Google Places Autocomplete**: Real-time address suggestions with dropdown selection for search box
- **Location Pin Markers**: Standard red map markers automatically placed at geocoded addresses
  - Drop animation on placement
  - Click marker to remove
  - Auto-removes on re-center or new address search
  - Zoom level 18 for optimal address viewing
- **Geocoding Fallback**: Manual address entry still supported via Google Geocoding API

## Business Address Features

- **Vendor Business Address Field**: New data field in vendor information panel
  - Admin mode: Editable textarea with Places Autocomplete validation
  - Viewer mode: Read-only display with interactive controls
  - Two-line mailing format: "Street\nCity, State ZIP"
  - Component parsing from Places API for consistent formatting
- **Address Navigation**: Map pin icon in viewer mode fills search box and navigates to address
- **Copy to Clipboard**: Document icon copies full address with toast notification "Address copied" (2000ms)
- **Address Validation**: Places API component parsing ensures properly formatted addresses

## Configuration & Setup

  - SECRET_KEY, GOOGLE_MAPS_API_KEY, MAP_ID, ADMIN_PIN, DATA_DIR
- **Google Maps API Libraries**: Added 'places' library to API initialization
- **Network Exposure**: Flask app now runs on 0.0.0.0:5000 for LAN/external access

## Cross-Platform Improvements

## Technical Implementation

### Badge System Architecture
- **Data Model**: Added `is_partner_vendor` and `is_featured_vendor` boolean fields to booth data
  - Existing: `is_event_staff`, `is_return_vendor`
  - All four fields normalized to boolean on load
- **Badge Creation**: Unified badge creation system in `draw-overlays.js`
  - Single `.booth-badge` base class with type-specific variants
  - Shared SVG definitions via `<use>` elements with `xlink:href`
  - Badge positioned at top-right polygon corner with transform offset
  - Z-index 300 for status badges (above labels and logo badges)
- **Visibility Management**: Three-point fix for zoom-based display
  1. Badge creation: Default hidden, check zoom before initial show
  2. `bringShapeToFront()`: Always apply zoom check when setting visibility
  3. `updateOverlapVisibility()`: Added `showBadgesAtZoom` check to badge visibility logic
- **State Tracking**: `shapeRecord.badgeType` stores active badge, `badgeOverlay` holds DomOverlay reference

### UI State Management
- **Panel Badge Updates**: `updateAllPanelBadges()` function shows/hides all four badge elements
  - Called from `updateReturnVendorUI()` and `updateEventStaffUI()`
  - Individual badge elements: `panelEventStaffBadge`, `panelPartnerVendorBadge`, `panelFeaturedVendorBadge`, `panelReturnVendorBadge`
- **Controls Integration**: 
  - Added `applyPartnerVendorUI()` and `applyFeaturedVendorUI()` functions
  - Checkbox change handlers auto-save on toggle
  - All four checkboxes read/written in assign/unassign logic
- **Scheduled Days State**: `updateScheduledDaysCheckboxStates()` function in `core-state.js`
  - Exported to `MCPP` namespace
  - Called from `who()` after auth check and `updateScheduledDaysUI()` after checkbox updates

### CSS & Layout
- **Flexbox Layout**: `.logo-badges-wrap` with `space-between` for side-by-side containers
- **Badge Grid**: CSS Grid 2×2 with `repeat(2, 40px)` and 6px gap
- **Toast Styling**: Centralized toast styles with consistent transitions (0.18s ease)
- **Legend Toggle**: CSS transitions on height, opacity, transform for smooth collapse
- **Label Exclusions**: Updated label selector to exclude `.scheduled-day-option` and `.badge-option`

### Address Features
- **Address Autocomplete Initialization**: Multiple strategies (immediate, delayed, map callback) ensure API readiness
- **Marker Management**: Scoped locationMarker variable with removeLocationMarker() helper exposed globally
- **Toast Positioning**: Anchor-based positioning system for context-aware notifications
- **State Management**: Added businessAddress, businessAddressViewer, businessAddressViewerWrap, businessAddressPin, businessAddressCopy to core-state.js
- **Data Persistence**: business_address field added to vendor data model with save/load/clear logic

## Search & Filtering Improvements

- **Booth ID Search**: Formatted booth IDs (B01, B02) now included in search index
  - Search by booth number or raw ID
  - Matches both formatted (B01) and raw (B1) forms
