# MCPAP Vendor Map — Release v0.4.0

Release date: 2026-01-27
QA: Manual QA completed by user

## Highlights

- **Logo Upload System**: Full support for uploading logo images (up to 100MB, any image format) or using URL
- **Revert-on-Back**: Unsaved edits automatically revert when backing out of booth editing
- **No Auto-Save**: Form fields only persist when explicitly clicking "Assign / Save" button
- **Enhanced Clear Details**: Clears all fields and map logo badge without auto-saving
- **Phone/Email Privacy Defaults**: Hidden checkboxes default to unchecked (visible by default)
- **Empty Field Handling**: Empty phone/email fields automatically treated as hidden on save

## Logo Management

### Logo Upload & URL Support
- **File Upload**: Upload logo images directly from admin panel (max 100MB, supports all common image formats)
- **Upload Endpoint**: `/api/upload-logo` (admin-only, secure filename handling)
- **URL Support**: Continue using external logo URLs as before
- **Mutual Exclusion**: Selecting a file clears URL field; entering URL clears file selection
- **Auto-Upload**: File uploads automatically when selected (no separate upload button)
- **Upload Directory**: Images stored in `static/uploads/` with unique filenames

### Logo Modal
- **Modal Interface**: Clean modal dialog for logo management
- **Preview**: Fixed 200×200px preview box at top of modal
- **Clear Button**: Clears URL, file input, and preview
- **Save Behavior**: Uses uploaded file URL if present, otherwise uses URL field
- **File Input Styling**: Custom-styled file input showing selected filename or "Select file – no file chosen"

## Edit Workflow Improvements

### Revert-on-Back Functionality
- **Snapshot System**: Captures booth state when opening for editing
- **Dirty Tracking**: Tracks unsaved changes to text fields, dimensions, rotation, badges, etc.
- **Automatic Revert**: Clicking "← Back" without saving restores last saved state
- **Visual Refresh**: Map, list, and form all update to reflect reverted state
- **Snapshot Management**: Only captures when switching to different booth (not on refresh)

### No Auto-Save
- **Explicit Save Only**: Changes only persist when clicking "Assign / Save" button
- **Badge Checkboxes**: No longer auto-save; update booth in memory and mark dirty
- **Rotation Changes**: Live preview on map but not persisted until Save
- **Form Fields**: All text inputs, selects, and checkboxes mark dirty but don't save
- **Enter Key Prevention**: Enter key in form fields doesn't trigger Save or activate buttons

### Clear Details Enhancement
- **Complete Clearing**: Clears all vendor fields, badges, scheduled days, and logo
- **Map Logo Badge**: Logo badge disappears from map immediately (no auto-save)
- **Form Refresh**: All form fields update to show cleared state
- **Placeholder Display**: Logo preview shows standard placeholder instead of broken image
- **Revert Support**: Backing out restores previously saved data (including logo)

## Phone & Email Privacy

### Default Visibility
- **Unchecked by Default**: Hidden checkboxes start unchecked (phone/email visible by default)
- **New Booths**: Phone and email default to visible (`phone_public: true`, `email_public: true`)
- **Existing Booths**: Missing privacy flags treated as visible (backward compatible)

### Empty Field Handling
- **Auto-Hide on Save**: Empty phone/email fields automatically set to hidden when saving
- **Checkbox Update**: Hidden checkboxes update immediately after save to reflect state
- **Viewer Display**: Empty fields never shown to viewers (hidden regardless of checkbox state)
- **Admin Display**: Admins always see phone/email fields for editing

## UI & UX Improvements

### Panel Behavior
- **Scroll Reset**: Edit panel scrolls to top when opening a booth
- **Form State**: Form fields properly reflect booth data when loading
- **Button Types**: All action buttons use `type="button"` to prevent form submission

### Visual Feedback
- **Save Toast**: "Changes saved successfully" notification appears after save
- **Upload Toast**: "Image uploaded successfully" when file upload completes
- **Logo Preview**: Standard placeholder SVG shown when no logo present

## Technical Implementation

### State Management
- **Booth Edit Snapshot**: Deep clone system for revert functionality
- **Dirty Flag**: Tracks whether current booth has unsaved changes
- **Snapshot Capture**: Only captures when switching booths (prevents overwrite on refresh)

### File Upload
- **Security**: Filename sanitization, size limits, extension validation
- **Storage**: Unique UUID-based filenames prevent conflicts
- **Serving**: Route to serve uploaded images from `static/uploads/`

### Form Validation
- **Phone Formatting**: Auto-formats phone numbers on blur/change
- **Empty Detection**: Checks for empty/whitespace-only fields
- **Privacy Logic**: Combines checkbox state with field content for final privacy setting

## Bug Fixes

- **Delete Booth Cleanup**: Properly removes logo badges, label overlays, and all map elements
- **Business Address Display**: Viewer wrapper only shows when not admin and address exists
- **Logo Modal Element**: Fixed "Logo modal element not found" by ensuring modal exists before script execution
- **Form Field Sync**: Clear Details now properly updates all form fields including logo preview
- **Map Logo Badge**: Logo badge updates immediately when clearing details (no page refresh needed)

## Configuration

- **Upload Directory**: `static/uploads/` created automatically
- **Max Upload Size**: 100MB limit for logo images
- **Allowed Formats**: All common image formats (jpg, png, gif, webp, svg, heic, etc.)

## Known Issues

- Mobile UI audit still pending from previous releases

## Migration Notes

- Uploaded logos stored in `static/uploads/` directory (ensure backups include this folder)
- `.gitignore` updated to exclude uploaded images
