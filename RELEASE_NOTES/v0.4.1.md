# MCPAP Vendor Map — Patch v0.4.1

Release date: 2026-02-27

## Mobile viewer

### Category badge positioning
- **Badge positioning fix**: Category (and status) badges now use a dedicated overlay that computes position in `draw()` from the booth path and nudge, so they stay correctly aligned in all scenarios (initial load, zoom, opening/closing booth detail, View on Map).
- **Category nudge**: Category badge horizontal nudge set to 23px so ribbons sit correctly inside booth polygons on mobile.
- **Removed legacy logic**: Old badge `setPosition` updates and related debug flags removed; badges self-position on every map draw.

### View on Map pulse animation
- **Pulse ring**: When using “View on Map” from the booth detail card, a pulse ring animates around the selected booth for 2 seconds.
- **Visibility**: Pulse uses the overlay pane and map canvas offset so it appears in the correct place; stroke and opacity increased so the ring is easy to see.
- **Two pulses**: Animation runs two gentle pulses (scale 1 → 1.06 → 1) over 2 seconds.
- **Fade-out**: Ring fades out over 0.35s at the end instead of disappearing abruptly.

### View on Map flow
- **Simplified flow**: Single pan to booth and one idle handler; no extra resize or double-idle, so map controls no longer jump when using View on Map.
- **Debug**: Pulse ring debug logging can be toggled with `DEBUG_PULSE_RING` (default on for now).

## Technical notes

- **createBadgePositionOverlay**: New overlay type that positions from path + corner index + nudge in `draw()`, keeping badges in sync with the map projection.
- **createPulseRingDom**: Pulse implemented as a DOM overlay in floatPane with a positioner overlay for correct coordinates; map canvas offset applied for mobile map div structure.
- **Pulse styling**: `.mv-pulse-ring-wrap` and `.mv-booth-pulse-ring` in `mobile-viewer.css`; fade-out via `.mv-pulse-ring-wrap--fade-out` and transition.
