# MCPAP Vendor Map — Hotfix v0.4.10

Release date: 2026-04-02

Hotfix on **v0.4.9**.

## Desktop (admin)

- **Logo modal Save / toasts**: Use shared **`MCPP.showToast`** from `ui-list.js` instead of a duplicate helper; avoids broken save feedback and aligns with the main toast (`#mcppToast`).
- **Vendor JSON export/import**: **`logo_embed`** (`mime` + base64 `b64`) for booths whose logo is stored under **`/static/uploads/`** (uploaded/cropped). Import **re-uploads** blobs so data works on a new server or Railway deploy. Plain external `https://` URLs are unchanged (not inlined).
- **Helpers**: `toast()`, `postLogoUpload()` consolidation; removed unused **`.save-toast`** CSS.

## Mobile (`/mobile`)

- **“View on Map” / Maps load**: Wait for **`google.maps`** before creating the map and before building **`LatLng`** so the map and polygons are not skipped when the API loads slowly (e.g. hosted on Railway).
- **`scheduleMapLayoutRecovery()`**: `resize` + delayed bumps + **`idle`** so tiles and overlays repaint after the detail sheet closes or the tab regains focus.
- **`visibilitychange`** / **`pageshow` (bfcache)**: Extra layout recovery when returning to the app.

## Technical

- Asset **cachebuster**: `v0.4.10`.

*Hotfix: reliability and admin portability; no API contract change beyond optional `logo_embed` on export files.*
