# MCPAP Vendor Map — Hotfix v0.4.12

Release date: 2026-04-02

Hotfix on **v0.4.11** (mobile Safari / WebKit map repaint).

## Mobile (`/mobile`)

- **`wakeMapForProgrammaticCamera`**: **`resize`** plus **`panBy(0, 0)`** before / when starting programmatic pan and zoom so the map canvas actually repaints; on **iOS Safari**, `setCenter` / `setZoom` could run without visible updates until the user panned.
- **`runWhenMapReadyForAnimation`**: Minimum **120ms** wait after entry before starting the pan; **resize + wake** before the callback; **650ms** idle fallback (was 500ms).
- **Vendor list → map** (`?view=map`): **`idle`** listener now has an **800ms** fallback if `idle` is delayed.
- **`scheduleMapLayoutRecovery` / `ensureMap`**: Dropped **`requestAnimationFrame`** in favor of **`setTimeout`** for resize bumps (consistent with other mobile paths).

## Technical

- Asset **cachebuster**: `v0.4.12`.
