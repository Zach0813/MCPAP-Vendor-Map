# MCPAP Vendor Map — Patch v0.4.2

Release date: 2026-03-08

## Mobile viewer

### Map controls on mobile
- **Center map**: New "Center map on booths" button on the map panel fits all vendor booths in view with padding and sets zoom after the map is idle.
- **Map labels**: New "Map labels" toggle switches between satellite and hybrid (labels) map type; button state reflects current mode.
- **Layout**: Controls sit at bottom-left above the tab bar, with solid dark styling consistent with desktop overlay buttons.

### Pan-then-zoom behavior
- **View on Map / list select**: When panning to a booth (from the list or "View on Map"), zoom is now applied only after the map finishes panning (idle). This avoids jumpy controls and a double-move feel.
- **Desktop**: Same idle-based zoom applied in `draw-overlays.js` when focusing a shape and in `ui-list.js` when selecting a booth from the list.

## Desktop / mobile breakpoint

### Tablet support
- **Breakpoint 768px → 1024px**: Mobile viewer is now used for viewports ≤1024px (phones and tablets). Desktop layout is used only for widths >1024px.
- **Redirect logic**: Main app and mobile viewer use matching breakpoints (1024 / 1025) so resizing doesn’t flip between layouts at different widths.
- **Viewport detection**: More robust mobile check using `innerWidth`, `clientWidth`, and `visualViewport` with fallbacks; multiple timeouts and `load` / `DOMContentLoaded` ensure redirect happens on small screens.

## Technical notes

- **mobile-viewer.js**: `centerMapOnBooths()`, `updateMapLabelsButton()`, `initMapControls()`; map type toggle stores state in `state.mapLabelsOn`.
- **mobile-viewer.css**: `.mv-map-controls`, `.mv-map-ctrl`, `.mv-map-ctrl-labels.on` for the new map buttons.
- **templates**: Map control buttons and SVGs added to `mobile-viewer.html`; index.html breakpoint and redirect script updated to 1024px and `goToMobile()`.
