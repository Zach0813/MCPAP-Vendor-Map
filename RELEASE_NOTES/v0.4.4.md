# MCPAP Vendor Map — Patch v0.4.4

Release date: 2026-03-08

## Railway deployment

- **railway.json**: Added Railway config-as-code so the start command and build settings live in the repo. No need to set the start command in the Railway dashboard.
- **Procfile**: `web: gunicorn -c gunicorn.conf.py app:app` for compatibility with platforms that use Procfiles.
- **runtime.txt**: Pins Python 3.11.9 for consistent builds.
- **README**: New "Deploy to Railway" section with step-by-step instructions and required environment variables (`GOOGLE_MAPS_API_KEY`, `ADMIN_PIN`, optional `MAP_ID`, `SECRET_KEY`).

Deploy by connecting the repo to Railway, setting the variables, and generating a domain. Railway sets `PORT` automatically; the app and Gunicorn config already use it.

## Technical notes

- `railway.json` uses `deploy.startCommand` and `build.builder: NIXPACKS`; restart policy on failure is enabled.
- Vendor data remains in the app filesystem (`data/vendors.json`); consider a volume or external storage for persistence across redeploys if needed.
