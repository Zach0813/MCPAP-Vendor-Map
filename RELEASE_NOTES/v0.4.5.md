# MCPAP Vendor Map — Patch v0.4.5

Release date: 2026-03-08

## Logo / image cropping

- **Center-crop slider**: When adding or editing a logo (upload or URL), a "Crop to center" slider (0–100%) zooms into the center of the image. No drag required; moving the slider shrinks the visible area toward the center. Save uses the cropped square (512×512 PNG) and uploads it.
- Cropper.js dependency removed; cropping is handled with a simple slider and canvas export.

## Business address & viewer

- **Business Address field**: Admin field changed from textarea to a single-line `<input type="text">` so Google Places Autocomplete works (API requires `HTMLInputElement`). Address from Autocomplete is formatted as a single line.
- **Viewer address**: In viewer mode, the Business Address box no longer focuses or highlights when clicked; it behaves like other read-only fields (no focus, no text selection).

## Event date filter

- **Desktop & mobile**: "Event Date" dropdown filters the map and list to one day (May 16 or May 17, 2026). Only booths scheduled for the selected day are shown; others are fully removed from the map (no overlap/collision with hidden booths).
- **Shared logic**: Scheduled day options and normalization live in `static/js/scheduled-days.js` and are used by both desktop and mobile views.

## Technical notes

- `scheduled-days.js` added; loaded before `core-state.js` and `mobile-viewer.js`.
- Logo modal: slider + `logo-crop-preview-wrap` with CSS transform for live preview; `getCenterCropCanvas()` in controls.js for canvas crop and upload on Save.
- Defensive check in Places Autocomplete init: only run when the element is an `HTMLInputElement`.
- Viewer address: `tabindex="-1"` and `mousedown` preventDefault to avoid focus.

---

*Easier logo cropping, reliable address autocomplete, and consistent event-day filtering across desktop and mobile.*
