# MCPAP Vendor Map — Hotfix v0.4.6

Release date: 2026-03-08

## Hotfix summary

- **Mobile Event Date dropdown**: Uses short numeric date format (e.g. `5/16/26`, `5/17/26`) so the selector fits nicely on small screens. Desktop keeps the full date string.
- **Railway data persistence**: Vendor data and uploaded logos no longer wiped on redeploy when using a Railway Volume. Set `DATA_DIR` to your volume mount path (e.g. `/data`); uploads are stored under the same directory when `DATA_DIR` is set. README updated with step-by-step Volume + variable setup.

## Changes

- **scheduled-days.js**: Added `dateShort` to options (`5/16/26`, `5/17/26`).
- **mobile-viewer.js**: Event Date dropdown displays `dateShort` for option labels.
- **app.py**: When `DATA_DIR` env is set, `UPLOAD_DIR` is `DATA_DIR/uploads` so both vendors.json and logos persist on the volume.
- **README.md**: "Persist vendor data (Railway Volume)" section with Volume creation and `DATA_DIR=/data` instructions.
