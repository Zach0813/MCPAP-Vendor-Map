# MCPAP Vendor Map — v0.4.7

Release date: 2026-03-08

## Logo crop behavior (fixes)

- **Crop from full image**: The crop tool no longer forces a square or “zoom to fill” the preview. At 0% you keep the full image; the slider (0–100%) shrinks a center rectangle from full size down to 10% of width/height. Non-square images are no longer auto-cropped to a square before your selection.
- **Output**: The saved image is still 512×512. The selected crop region is drawn centered on the canvas; any letterboxing is **transparent** (no stretching to fill the square).
- **WYSIWYG preview**: The modal preview now shows exactly what will be saved—the same center rectangle, scaled to fit in the preview box with transparent edges when the crop isn’t square—so there’s no mismatch between preview and saved image.
- **Debug**: “Show crop region on image” draws a red rectangle on the original image so you can confirm the exact pixel region that will be saved.

## Technical notes

- **controls.js**: Replaced zoom-based square crop with `getLogoCropRect()` (center rect: `cropW = w*(1-0.9*pct)`, `cropH = h*(1-0.9*pct)`). Preview uses a 280×280 canvas that redraws the same crop; save uses the same rect on a 512×512 canvas with transparent background. Debug button uses `getCenterCropRect()` and shows crop dimensions (e.g. `crop: 377×245`).
