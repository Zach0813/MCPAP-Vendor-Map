# MCPAP Vendor Map — v0.4.8

Release date: 2026-04-01

## Data durability (patch)

- **`vendors.json` write path**: After writing booth data, the server now **flushes** the file buffer and **`fsync`s** the file descriptor so contents are pushed to the filesystem (and volume) before the HTTP response returns.
- **Why**: On hosts like Railway with attached volumes, relying on the default close behavior can occasionally leave recent saves only in the page cache; a restart or crash could lose the last write. Syncing after each save reduces that window.

## Technical notes

- **`app.py` — `save_booths()`**: `f.flush()` then `os.fsync(f.fileno())` immediately after `json.dump(...)`.
